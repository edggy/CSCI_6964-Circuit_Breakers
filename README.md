# CSCI 6964 – Circuit Breakers

Final project for CSCI 6964.


## Overview
In this lab, you will use the provided notebook to study how AI-assisted Trojan workflows are structured
and how defenders can analyze suspicious RTL modifications. You will work with **provided sample
Trojaned outputs** (or instructor-supplied artifacts) to extract trigger/payload descriptions, compare
vulnerability categories, and validate basic syntax and test behavior.

### Learning objectives
- Explain common hardware Trojan concepts (trigger, payload, stealthiness) using concrete
RTL examples.
- Interpret a vulnerability taxonomy (e.g., T1–T4) and map examples to each category.
- Run basic validation checks (syntax/lint/simulation where applicable) and summarize
results.
- Design a simple defensive heuristic/validator and evaluate it on sample artifacts.

### Prerequisites
You should be comfortable reading Verilog RTL and running basic tooling (e.g., iverilog). No prior
hardware security experience is required.

## Setup
1. Open `Hardware_Trojan_Insertion_Lab.ipynb` in Colab or local Jupyter.
2. Install any required dependencies (the notebook contains setup cells).
3. Ensure `iverilog` is available if you plan to run syntax compilation checks.

`trojaned_outputs_sample`/).
